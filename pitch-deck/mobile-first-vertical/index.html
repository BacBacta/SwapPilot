03 / 12
Solution

One flow that optimizes & explains.

SwapPilot separates “best raw output” from “best executable” and documents the decision in a receipt (whyWinner, warnings, mode rationale).

BEQ (Best Executable Quote)
Optimizes for executability first — then for output — instead of only chasing the highest quote.
Decision receipt
Every quote returns a receiptId with: whyWinner · warnings · ranking rationale · normalization assumptions.
Risk modes
SAFE / NORMAL / DEGEN switch the BEQ weights — while the receipt keeps the reasoning transparent.
Clear tradeoffs. Verifiable claims only.

04 / 12
Product

How a swap happens

Four steps — mapped to real API calls.

1) Quote
POST /v1/quotes — fan-out to multiple providers in parallel (BSC chainId 56).
2) Rank
Return rankedQuotes (BEQ) + bestRawQuotes + beqRecommendedProviderId.
3) Explain
GET /v1/receipts/:id — whyWinner + warnings + assumptions in a single artifact.
4) Execute
Use /v1/build-tx when supported (ready-to-sign tx + approvalAddress). Otherwise open the provider via deepLink.
Simple mental model: quote → rank → explain → execute.
